# SP4 — GTK4 GUI — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** A native GTK4 config window (`zclicker-gui`) that wraps the `zclicker` CLI via spawn, exposing mode/click/interval/triggers/output/device/suppress, with capture-to-bind and live status.

**Architecture:** Second build target, same `0.17-dev` toolchain, GTK4 linked as a C lib via `@cImport`. Pure `command.zig` builds the argv (tested); `capture.zig` reads the next evdev code (reuses `platform/linux.zig`); `main.zig` is GTK glue that spawns/stops the CLI with `GSubprocess` and reads `zclicker --print-env` to show the auto backend.

**Tech Stack:** Zig `0.17.0-dev.305`, GTK4 4.22 (`@cImport <gtk/gtk.h>`), GLib `GSubprocess`.

**GTK-from-Zig gotchas (apply throughout):**
- `g_signal_connect` is a C **macro** → not available via `@cImport`. Use `c.g_signal_connect_data(obj, "signal-name", @ptrCast(&callback), user_data, null, 0)`.
- Signal callbacks must be `callconv(.c)` functions; cast with `@ptrCast` to the `GCallback` arg.
- Pass widget pointers to callbacks via a heap-allocated `Ui` struct used as `user_data`.
- `@cImport` once, in `src/gui/c.zig`; everything else imports that.
- Build/iterate constantly — GTK translation specifics are confirmed against the compiler, not memorized.

const std = @import("std");

pub const Mode = enum { hold, toggle };
pub const Click = enum { left, right, middle };
pub const Output = enum { auto, uinput, ydotool };

pub const Config = struct {
    interval_ms: u32 = 50,
    mode: Mode = .hold,
    click: Click = .left,
    /// Trigger codes already resolved to evdev numbers (e.g. 275, 276).
    codes: []const u16 = &.{},
    output: Output = .auto,
    device: []const u8 = "", // empty = auto
    suppress: bool = false,
};

/// Build argv (argv[0] = bin_path). Caller owns the returned slice and each string;
/// use an arena to free them all at once.
pub fn buildArgv(arena: std.mem.Allocator, bin_path: []const u8, cfg: Config) ![]const []const u8 {
    var list: std.ArrayList([]const u8) = .empty;
    try list.append(arena, bin_path);
    try list.append(arena, "-i");
    try list.append(arena, try std.fmt.allocPrint(arena, "{d}", .{cfg.interval_ms}));
    try list.append(arena, "--mode");
    try list.append(arena, @tagName(cfg.mode));
    try list.append(arena, "--click");
    try list.append(arena, @tagName(cfg.click));
    if (cfg.codes.len > 0) {
        var buf: std.ArrayList(u8) = .empty;
        for (cfg.codes, 0..) |code, idx| {
            if (idx != 0) try buf.append(arena, ',');
            try buf.writer(arena).print("{d}", .{code});
        }
        try list.append(arena, "-b");
        try list.append(arena, buf.items);
    }
    if (cfg.output != .auto) {
        try list.append(arena, "--output");
        try list.append(arena, @tagName(cfg.output));
    }
    if (cfg.device.len > 0) {
        try list.append(arena, "-d");
        try list.append(arena, cfg.device);
    }
    if (cfg.suppress) try list.append(arena, "--suppress");
    return list.items;
}

test "defaults" {
    const t = std.testing;
    var arena = std.heap.ArenaAllocator.init(t.allocator);
    defer arena.deinit();
    const argv = try buildArgv(arena.allocator(), "zclicker", .{});
    try t.expectEqualStrings("zclicker", argv[0]);
    try t.expectEqualStrings("-i", argv[1]);
    try t.expectEqualStrings("50", argv[2]);
    try t.expectEqualStrings("--mode", argv[3]);
    try t.expectEqualStrings("hold", argv[4]);
    try t.expectEqualStrings("--click", argv[5]);
    try t.expectEqualStrings("left", argv[6]);
    try t.expectEqual(@as(usize, 7), argv.len); // no -b/--output/-d/--suppress by default
}

test "full options" {
    const t = std.testing;
    var arena = std.heap.ArenaAllocator.init(t.allocator);
    defer arena.deinit();
    const codes = [_]u16{ 275, 276 };
    const argv = try buildArgv(arena.allocator(), "/bin/zclicker", .{
        .interval_ms = 30, .mode = .toggle, .click = .right,
        .codes = &codes, .output = .ydotool, .device = "/dev/input/event6", .suppress = true,
    });
    // find the -b value
    var saw_buttons = false;
    for (argv, 0..) |a, idx| {
        if (std.mem.eql(u8, a, "-b")) { try t.expectEqualStrings("275,276", argv[idx + 1]); saw_buttons = true; }
    }
    try t.expect(saw_buttons);

    var arena = std.heap.ArenaAllocator.init(t.allocator);
    defer arena.deinit();

### Task 3: `capture.zig` — read the next trigger code

**Files:**
- Create: `src/gui/capture.zig`

- [ ] **Step 1: Implement** — reuse the engine's Linux helpers (`../platform/linux.zig`):
```zig
const std = @import("std");
const lx = @import("../platform/linux.zig");

const MAX_EVENT_NODES: usize = 64;

/// Block until the user presses any key/button on any real input device, then return
/// its evdev code. Skips virtual devices. Needs read access to /dev/input (input group).
pub fn captureNext() !u16 {
    var fds: [MAX_EVENT_NODES]std.posix.pollfd = undefined;
    var paths_open: [MAX_EVENT_NODES]std.posix.fd_t = undefined;
    var count: usize = 0;
    defer for (paths_open[0..count]) |fd| lx.closeFd(fd);

    var n: usize = 0;
    while (n < MAX_EVENT_NODES and count < MAX_EVENT_NODES) : (n += 1) {
        var pathbuf: [32]u8 = undefined;
        const path = std.fmt.bufPrintSentinel(&pathbuf, "/dev/input/event{d}", .{n}, 0) catch continue;
        const fd = lx.openRdonlyNonblock(path) catch continue;
        // Only key-capable, non-virtual devices.
        if (isVirtual(fd) or !hasAnyKey(fd)) { lx.closeFd(fd); continue; }
        paths_open[count] = fd;
        fds[count] = .{ .fd = fd, .events = std.posix.POLL.IN, .revents = 0 };
        count += 1;
    }
    if (count == 0) return error.NoDevices;

    while (true) {
        _ = try std.posix.poll(fds[0..count], -1);
        var i: usize = 0;
        while (i < count) : (i += 1) {
            if ((@as(i16, fds[i].revents) & @as(i16, std.posix.POLL.IN)) == 0) continue;
            var buf: [64]lx.InputEvent = undefined;
            const bytes = std.posix.read(fds[i].fd, std.mem.sliceAsBytes(buf[0..])) catch continue;
            const ne = bytes / @sizeOf(lx.InputEvent);
            for (buf[0..ne]) |ev| {
                if (ev.type == lx.EV_KEY and ev.value == 1) return ev.code; // first press
            }
        }
    }
}

const InputId = extern struct { bustype: u16, vendor: u16, product: u16, version: u16 };
fn isVirtual(fd: std.posix.fd_t) bool {
    var id: InputId = undefined;
    const rc = std.os.linux.ioctl(fd, lx.ior('E', 0x02, @sizeOf(InputId)), @intFromPtr(&id));
    if (@as(isize, @bitCast(rc)) < 0) return false;
    return id.bustype == 0x06;
}
fn hasAnyKey(fd: std.posix.fd_t) bool {
    var keybits: [lx.KEY_MAX / 8 + 1]u8 = @splat(0);
    const rc = std.os.linux.ioctl(fd, lx.ior('E', 0x20 + @as(u32, lx.EV_KEY), keybits.len), @intFromPtr(&keybits));
    return @as(isize, @bitCast(rc)) >= 0;
}
```

- [ ] **Step 1: Define a `Ui` struct** holding pointers to every widget that a callback needs (`interval_spin`, `mode_hold/mode_toggle` checkbuttons or a single toggle, `click_dropdown`, `output_dropdown`, `device_entry`, `suppress_check`, `status_label`, `start_button`, the trigger list container and a dynamic `std.ArrayList(u16)` of captured codes, plus `?*c.GSubprocess child`). - [ ] **Step 2: Build the window** in `onActivate`: a vertical `gtk_box`; rows built with `gtk_box` + `gtk_label` + the control:
  - interval: `c.gtk_spin_button_new_with_range(1, 100000, 1)`, set value 50.
  - mode: two `gtk_check_button_new_with_label` in a group (`gtk_check_button_set_group`) "Segurar"/"Alternar", or a single toggle labeled "Alternar (toggle)".
  - click: `c.gtk_drop_down_new_from_strings(&[_][*c]const u8{ "left", "right", "middle", null })`.
  - output: `gtk_drop_down_new_from_strings` with `{ "auto", "uinput", "ydotool", null }`.
  - device: `gtk_entry_new()` with placeholder.
  - suppress: `gtk_check_button_new_with_label("Suprimir voltar/avançar (só mouse)")`.
  - triggers: a horizontal box listing current codes as labels + a "Capturar" button.
  - status: `gtk_label_new("parado")`.
  - start: `gtk_button_new_with_label("Iniciar")`.
  Append all to the box; `gtk_window_set_child`.

- [ ] **Step 3: "Capturar" callback** → call `capture.captureNext()` (it blocks; acceptable for v1 — note it freezes the UI briefly; a v2 could thread it), append the code to the `Ui.codes` list, and refresh the triggers row. Map known codes back to friendly names via a small table (BTN_SIDE→"4", etc.), else show the number.

- [ ] **Step 4: Read widgets → `command.Config`** in a helper: spin value (`gtk_spin_button_get_value_as_int`), selected dropdown index (`gtk_drop_down_get_selected`) → enum, entry text (`gtk_editable_get_text`), check states (`gtk_check_button_get_active`), codes from `Ui.codes`.

- [ ] **Step 5: Start/Stop callback:**
  - If no child running: build argv via `command.buildArgv` (arena), resolve the binary path (`$ZCLICKER_BIN` → `/proc/self/exe` dir + "/zclicker" → "zclicker"), spawn with `c.g_subprocess_newv(argv_c_ptr, c.G_SUBPROCESS_FLAGS_STDERR_PIPE, &gerr)`. Store child; set button label "Parar", status "rodando". Use `c.g_subprocess_wait_async(child, null, &onChildExit, ui)` to reset on exit (and read stderr via `g_subprocess_get_stderr_pipe` to surface errors on fast nonzero exit).
  - If child running: `c.g_subprocess_send_signal(child, std.posix.SIG.TERM)`; the wait_async callback resets the UI.
  - argv conversion: build a null-terminated `[*c]const [*c]const u8` from the Zig argv (each string must be NUL-terminated — use the arena to dup with sentinel via `std.fmt.allocPrintSentinel`/`arena.dupeZ`).

- [ ] **Step 6: Populate the output "auto" hint** by running `zclicker --print-env` once at startup (via `g_subprocess` sync or `std.process` — simplest: spawn and read stdout) and parsing `auto-output:`/`os:`/`session:`; show it next to the output dropdown (e.g. update a label "auto → uinput"). If this is fiddly, a static "auto" label is acceptable for v1 — note it.

- [ ] **Step 7: Verify** `zig build -Dgui` compiles/links; `zig build gui` opens the window. Manual smoke test: set options, Capturar binds a button, Iniciar spawns (status → rodando), Parar stops. (Real clicking requires /dev/uinput access.)
